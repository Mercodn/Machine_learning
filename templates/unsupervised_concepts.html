🔍 Unsupervised Learning - Basic Concepts

Understanding clustering and pattern discovery without labeled data

What is Unsupervised Learning?

Unsupervised learning is a category of machine learning algorithms that discover hidden patterns and structures in unlabeled data. Unlike supervised learning, where data points have labels or target values, unsupervised learning algorithms work with data where the output or grouping is unknown.

Key Difference:

Supervised Learning: Data has labels → Predict labels for new data

Unsupervised Learning: Data has no labels → Discover hidden patterns and groups

Applications of Unsupervised Learning

Customer Segmentation

Group customers by purchasing behavior, demographics, and preferences

Image Compression

Reduce image file size by clustering similar colors and pixels

Gene Sequence Analysis

Group DNA sequences to identify similar genetic patterns

Document Clustering

Organize large document collections by topic and content similarity

Anomaly Detection

Identify unusual patterns or outliers in data for fraud detection

Social Network Analysis

Discover communities and groups of users with similar interests

Clustering: The Main Unsupervised Learning Task

Clustering is the process of grouping similar data points together into clusters. Each cluster contains data points that are more similar to each other than to points in other clusters.

Goals of Clustering:

  • Maximize Intra-cluster Similarity: Points in the same cluster should be as similar as possible
  • Minimize Inter-cluster Similarity: Points in different clusters should be as different as possible
  • Discover Meaningful Groups: Clusters should represent natural groupings in the data
  • Reduce Dimensionality: Organize high-dimensional data into meaningful lower-dimensional groups

K-Means Clustering Algorithm

K-Means is one of the most popular unsupervised learning algorithms. It partitions data into K clusters by minimizing the distance between points and their cluster centroids.

What is a Centroid?

A centroid is the center point of a cluster. It represents the average position of all points in that cluster. In 2D space, if a cluster contains points (x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ), the centroid is at ((x₁+x₂+...+xₙ)/n, (y₁+y₂+...+yₙ)/n).

Distance Metrics: Euclidean Distance

K-Means typically uses Euclidean distance to measure how far a data point is from a centroid.

For two points A(x₁, y₁) and B(x₂, y₂):

distance = √[(x₂-x₁)² + (y₂-y₁)²]

This formula extends to higher dimensions and is the most common distance metric in machine learning.

K-Means Algorithm: Step-by-Step Process

Step 1: Initialize K Centroids

Randomly select K initial centroids from the data or choose them randomly in the feature space.

Step 2: Assign Points to Nearest Centroid

For each data point, calculate its distance to all K centroids and assign it to the cluster with the nearest centroid.

Step 3: Recalculate Centroids

Compute the new centroid for each cluster as the mean (average) of all points currently in that cluster.

Step 4: Check for Convergence

If centroids have moved significantly, repeat steps 2-3. If centroids are stable (convergence), the algorithm has finished.

Iterative Process and Convergence

K-Means is an iterative algorithm, meaning it repeats the same steps until a stopping condition is met. Typically:

  • Convergence: The algorithm stops when centroids stop moving (or move less than a threshold)
  • Maximum Iterations: Set a maximum number of iterations to prevent infinite loops
  • Stability: The final cluster assignments represent local optimal groupings

Variance and Cluster Quality

The quality of clustering is often measured by within-cluster variance, also called inertia:

Inertia = Sum of squared distances from each point to its centroid

Lower inertia → More compact, better-defined clusters

As K-Means iterates, variance generally decreases, indicating that clusters are becoming more cohesive and well-separated.

Choosing the Right K

One challenge in K-Means is deciding how many clusters (K) to create. Common methods:

Elbow Method

Plot inertia vs. K value. The "elbow" point (where inertia decreases more slowly) suggests the optimal K.

Silhouette Analysis

Measures how similar each point is to its own cluster compared to other clusters. Score ranges from -1 to 1.

Domain Knowledge

Use business or domain expertise to decide meaningful K value.

Gap Statistic

Compares the within-cluster variation with what would be expected for random data.

Real-World Example: Customer Segmentation

Imagine a retail company with 1,000 customers. By applying K-Means with K=3, they might discover:

Cluster 1 - Budget Shoppers: Low spending, buy few products, respond to promotions

Cluster 2 - Regular Customers: Moderate spending, loyal, good product diversity

Cluster 3 - Premium Clients: High spending, buy many products, value quality

These insights enable targeted marketing strategies for each segment.

Advantages and Disadvantages

Advantages

  • Simple and easy to understand
  • Fast computational time
  • Scales well to large datasets
  • Flexible with different data types

Disadvantages

  • Must specify K in advance
  • Sensitive to initial centroid placement
  • Assumes spherical clusters
  • Can get stuck in local optima
